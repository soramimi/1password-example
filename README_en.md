# 1password-example

A sample project that securely retrieves secrets (API keys, etc.) from 1Password using Desktop App Integration in a C/C++ application.

## Overview

This project wraps the official 1Password Go SDK as a shared library via CGo, which is then called from C++. Secrets are never hardcoded in source files — they are fetched at runtime from a 1Password Vault.

```
┌─────────────────────────────────────────────────────┐
│  myapp (C++)                                        │
│    ↓ calls GetAPIKEY()                              │
│  libapikey.so (Go + CGo)                           │
│    ↓ resolves secret via 1Password SDK              │
│  1Password desktop app                              │
│    ↓                                               │
│  1Password Vault (op://API_KEY/Example/credential) │
└─────────────────────────────────────────────────────┘
```

## Directory Structure

```
1password-example/
├── Makefile              # Top-level build and run script
├── libapikey/            # Go shared library
│   ├── main.go           # Implementation of GetAPIKEY / FreeString
│   ├── libapikey.h       # Export header generated by CGo
│   ├── go.mod            # Go module definition
│   └── Makefile          # Build script for libapikey.so
└── main/                 # C++ main application
    ├── main.cpp          # Entry point
    ├── main.h            # Local header
    └── Makefile          # Build script for myapp
```

## Prerequisites

- [1Password desktop app](https://1password.com/downloads/) installed and signed in
- A Vault named `API_KEY` in 1Password containing an item called `Example` with a `credential` field holding the secret
- Go 1.26.4 or later
- GCC / G++
- Make

## Build

```bash
make
```

This runs the following steps in order:

1. Build the Go shared library (`libapikey.so`) inside `libapikey/`
2. Build the C++ application (`myapp`) inside `main/` and link it against `libapikey.so`

## Run

```bash
OP_ACCOUNT_NAME="Your Account Name" make run
```

Set the `OP_ACCOUNT_NAME` environment variable to the display name of your 1Password account (e.g., `John Doe`).

The retrieved secret will be printed to standard output.

## API Reference

### `GetAPIKEY(account_name *C.char) *C.char`

Retrieves a secret from 1Password via Desktop App Integration.

| | Description |
|---|---|
| Argument | `account_name` — display name of the 1Password account |
| Return value | Pointer to a secret string allocated with `C.malloc` |
| On error | Panics on client initialization failure or secret resolution failure |

> **Note**: Always call `FreeString()` to release the returned pointer after use.

### `FreeString(s *C.char)`

Frees the memory of the C string returned by `GetAPIKEY()`.

## Security

- Secrets are never hardcoded in source code or configuration files; they are fetched at runtime from a 1Password Vault.
- After printing the secret, `myapp` immediately zeroes the memory with `memset()` to reduce the risk of exposure through memory dumps.
